🎨 EchoHire - Architecture & Diagrams BranchThis branch contains the comprehensive design assets and the Software Architecture Document (SAD) for EchoHire. The architecture was designed to handle the multi-modal nature of the platform, ensuring high performance for real-time AI interactions and secure code execution.🏗️ Architectural Choice: Hybrid Micro-LayeredWe selected a Hybrid Architecture as the most appropriate style for this project.Layered (N-Tier) Style: Used internally within each module to maintain a clear separation between presentation, logic, and data.Microservices Style: Used at the system level to decompose the project into independent functional blocks (Interview, Coding, Career, Analytics).Justification for this Choice:Independent Scaling: The Real-Time Coding Environment requires significant CPU resources compared to the Resume Scanner. Microservices allow these to scale separately.Strict Security: The Coding Sandbox is isolated to prevent user-submitted code from compromising the main AI Interviewer or system databases.Fault Tolerance: A failure in the LinkedIn Profile Optimizer will not interrupt an ongoing Live Voice Interaction.📁 Diagram Repository1.
